# 全项目轻量化审计

审计日期：2026-07-02

## 结论

当前 `wallpaper/` 保持无构建、无本地服务、无 Electron 的直接运行结构。发布版 1.0 运行包共 18 个文件、3,819,221 bytes（3.642 MiB）；默认页面启动载荷 871,294 bytes（0.831 MiB）。相对此前只增加项目标题与简介的发布版标识。最大文件是只用于 Wallpaper Engine 列表展示的预览图、仅在骷髅预设按需读取的点云和本地 Three.js，三者均有明确用途，不是孤儿文件。

本轮没有合并模块或压缩业务源码。该做法虽然能减少少量文件/字节，却会降低故障定位和上游映射能力，对 CEF 运行性能没有实质收益。

## 审计范围

- 运行入口、2 个 CSS、10 个业务模块、本地 Three.js、3 个资产和 `project.json`。
- 7 个验证/诊断脚本及其 PowerShell/JavaScript 语法。
- 6 份已有架构、范围、映射、增强、验证和交接文档。
- `project.json` 全部 DIY 属性的归一化与运行时消费链路。
- `index.html` 全部脚本/样式引用，以及 `assets/` 全部资源引用。

`upstream-mineradio/` 只作为上游真值，不计入可发布壁纸运行包，也未修改。

## 已落地优化

1. 骷髅点云从启动预读改为选择骷髅预设时按需读取，默认启动少读约 1 MiB 二进制并避免提前创建对应 GPU 几何体。
2. 操作日志关闭时不再读取历史日志、持久化日志或构造高频 timeline 原始采样对象。
3. 歌词缓存改为单次载入内存复用，并限制为 96 首或约 1.2 MB。
4. 歌词样式只在字体、字重、字距或行高变化时重绘，不再每帧创建样式对象和签名字符串。
5. 歌词配色只在媒体调色板或相关设置变化时更新，不再每帧创建多组 `THREE.Color`。
6. 音频频段、节拍分析和渲染帧对象改为帧间复用；公开 `snapshot()` 仍返回独立副本，避免破坏外部读取语义。
7. 主渲染直接只读当前视觉状态，并复用相机音频对象，不再每帧复制整份属性对象。
8. 预览 PNG 进行无损重压缩，像素逐点一致，文件从 1,974,708 bytes 降至 1,899,607 bytes。

## 自动约束

`tests/regression.test.js` 现在会阻止以下退化：

- `index.html` 引用缺失，或 `wallpaper/js` 模块未加载/重复加载；
- `assets/` 出现孤儿资源；
- DIY 属性未进入 `properties.js`，或归一化后没有运行时消费者；
- 壁纸运行包超过 5 MiB；
- 默认启动载荷超过 1.2 MiB；
- 骷髅点云重新变成启动预读；
- 主循环重新复制视觉快照，或音频帧不再复用；
- 网易云 ID、权威 timeline、seek 二分定位和歌词视觉切行契约失效。

## 保留项与理由

- `three.r128.min.js`：唯一运行时图形依赖，已是压缩版；替换版本会改变 shader/CEF 兼容性，不属于无风险减重。
- `skull-decimation-points.bin`：暗魂骷髅预设的实际几何数据；已按需加载，继续删减会改变视觉效果。
- `preview-3.0.png`：Wallpaper Engine 项目预览，不进入页面启动链路；仅做无损压缩，不降分辨率或有损量化。
- 各业务模块内部少量 `clamp` 等纯函数：合并后只会引入跨模块依赖，节省不足以抵消耦合。

## 验收边界

静态与逻辑回归只能证明结构、状态机和资源契约。8 个预设、镜头差异、歌词渲染和骷髅按需加载仍以 `tests/wpe-visual-regression.ps1` 的 Wallpaper Engine CEF 截图为最终视觉证据；真实网易云拖动同步仍以桌面媒体 timeline 和操作日志为最终证据。
