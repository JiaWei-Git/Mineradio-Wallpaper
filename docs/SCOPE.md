# 迁移范围

## 本阶段包含

- 原版 7 个视觉预设及其默认参数。
- 原版封面粒子、星河、骷髅资源、相机和切歌转场。
- 原版鼠标旋转、悬停及镜头交互中适用于壁纸的部分。
- Wallpaper Engine 镜头观察中心、观察高度、前后位置和仰俯角 DIY 控制。
- Wallpaper Engine 音频频谱、媒体元数据、封面、播放状态适配。
- 基于本地媒体播放历史的 3D 歌单架。
- 原版中有视觉意义且能静态表达的 DIY 设置。
- 基于媒体元数据、网易云公开歌词接口和 LRCLIB 备用源的同步歌词、缓存、降级与 MINERADIO 舞台歌词视觉。

## 本阶段不包含

- Mineradio 播放器、搜索、账号、收藏、网络 API 和更新系统。
- Electron 主进程、Node.js 服务、独立桌面歌词窗口。
- `wallpaper/` 运行时内的 BetterNCM、InfLink-rs、DLL 注入和其他播放器插件桥接。

## 1.0 发布层

- 仓库外层可以提供 Wallpaper Engine 本地项目安装器和可选网易云增强安装入口。
- 增强入口不得被壁纸自动调用；必须由用户主动运行并确认未签名 DLL 注入风险。
- BetterNCM 只从固定官方 Release 下载并校验 SHA-256，不分发来源无法映射到公开源码的本机 `msimg32.dll`。
- 经修改的 InfLink-rs 包必须同时提供原始版本、完整对应源码、修改补丁、许可证和 SHA-256。
- 摄像头手势控制。

## 兼容原则

原版音频输入来自播放器内部 Web Audio 节点；Wallpaper Engine 只能提供系统频谱。视觉算法保持原版，输入通过适配层归一化。任何无法完全一致的行为都记录在 `UPSTREAM_MAP.md`，不隐藏差异。

## 完成度定义

- “已迁移”：目标功能在 Wallpaper Engine 中可运行并有回归验证。
- “一比一”：必须把上游对应函数、默认参数、状态机和逐帧截图全部交叉验证后才能标记。

当前 8 个预设属于“已迁移”。3D 歌单架已接入媒体历史，但视觉布局是兼容实现；完整原版歌单架、所有镜头状态和悬停转场尚未一比一迁移。
